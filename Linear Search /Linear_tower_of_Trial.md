# **Linear Search Challenges (Levels)**

## 🏰 **Level 1: The Trial of the Seeker**
### **Problem:**
You are given an unsorted array of `n` integers where all elements are positive except for **one unique negative number**. However, the position of this negative number is unknown. You are only allowed to use a **modified linear search** to find the index of this negative number, but with the following constraints:

- You are **not allowed to use any built-in comparison operators** like `<`, `>`, or `==`.
- You can only use **bitwise operations, arithmetic operations, and indexing**.
- Your task is to implement a solution that **returns the index** of the negative number using linear search under these constraints.
- Further, your solution should minimize the number of array accesses.

---

## ⚔️ **Level 2: The Search of the Eternal Prime**
### **Problem:**
You are given an **infinite-length data stream** of integers arriving one-by-one (you can only access the next element, and you **cannot go backward**). The stream contains:

- All integers are **non-negative**, except there is **one and only one prime number** that occurs exactly **once** in the entire stream.
- All other numbers are either **composite** or **zero**, and may repeat any number of times.

You need to **detect the unique prime number** using a linear-search-like method. However:

### **Constraints:**
1. You can only use **O(1) space** (i.e., constant memory — no full array or list storage).
2. You can make **only one pass** through the stream.
3. You may **not use any built-in prime-checking functions**.
4. The stream can be potentially **infinitely long**, so you cannot assume all numbers will eventually appear.

---

### **Follow-Up Challenge:**
Explain how your approach ensures that the number of array accesses is minimized, and analyze the time complexity in terms of `n`.

---

## 🏆 **Level 3: The Trial of the Corrupted Oracle**
### **Problem:**
You are given an **hostile memory environment**. You are given an array of `n` elements where:

- Exactly **one element is corrupted** — it lies about its value when accessed (returns a wrong number).
- The remaining `n-1` elements are correct.
- You need to **search for a target value `T`** using **only linear search**.
- You do **not know** which element is corrupted.
- The corrupted element's value changes each time you access it — it **mutates adversarially** to defeat your logic.

### **Constraints:**
1. You may only access each index **at most twice**.
2. You must return the **index** of `T` if it exists, or `-1` otherwise.
3. You must **tolerate** the presence of this corrupt element — meaning, you must **guarantee correctness despite its lies**.
4. You may not sort, hash, or copy the array — **pure linear scan only** with minimal extra memory (`O(1)`).

---

## 🌟 **Level 4: The Quest for the Chaotic Sequence**
### **Problem:**
You are given an **infinite, non-repeating, pseudo-random stream** of integers. The stream is **generated by an unknown source** following a **deterministic chaotic function** (e.g., a sequence defined by a **non-linear recurrence relation**), but its exact formula is **hidden** from you.

The stream is **random-looking**, but there is a **special property**: There exists a **unique, deterministic subsequence** within the infinite stream that **matches a given target sequence `S`** of length `k`. However, the catch is that you don’t know the position of this subsequence, nor the specific structure of the stream, except for the following:

- The stream is **generated** in such a way that the subsequence **appears once and only once**, but you cannot rely on any classical random detection methods like hashing or sorting.
- **Your task is to locate this subsequence `S` of length `k` in the infinite stream using only linear search** (one pass at a time).

### **Constraints:**
1. **You have no access to the stream's global structure**. You can only look at the sequence as it arrives — meaning you cannot backtrack, jump, or randomly access any earlier elements.
2. **No additional memory** for caching previous values (pure O(1) space).
3. You can **only make one pass** through the stream — no revisiting of elements.
4. The stream **may change its generation rules after each subsequence is found**, but the subsequence you're looking for is **deterministically generated**.

---

### **Bonus Challenge:**
- Explain how you would **optimize the search process** if the subsequence `S` were guaranteed to appear at least once in **every finite window of size `k`**, regardless of how the stream changes.

---

## 🔮 **Max-Level Linear Search Problem: The Infinite Labyrinth of Truth**
### **Problem:**
You are working in a **hostile memory environment** with an **infinite data stream** where the stream consists of values that follow **a chaotic, hidden deterministic function**. You must locate a specific **deterministic subsequence** in this stream that **matches a given pattern `P`** (length `k`).

### **Constraints:**
1. You are only allowed **one linear scan** (no backtracking, no additional data structures).
2. The subsequence **appears once in a highly irregular fashion** — it is part of a non-linear, chaotic, yet deterministic pattern.
3. You cannot rely on randomness or sampling — all access to the stream must be **directly sequential**.
4. **You cannot cache or use additional memory**, and you must adapt your search to **an evolving, unpredictable environment** where the sequence's behavior may change unpredictably, but still deterministic.

---

### **This is a theoretical problem designed to test the boundaries of algorithmic search, chaos theory, and understanding infinite data streams.**
